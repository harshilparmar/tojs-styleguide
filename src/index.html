Brand Manual & Style Guide

Version 2.0

The Toronto JS Brand Manual & Style Guide is a living document that describes the style we use at Toronto JS. It is intended to be a reference for current and future contributors to the Toronto JS community who wish to develop or create anything using Toronto JS branding.

Using the fonts ðŸ”—

Montserrat - Medium/extra bold

Montserrat is the main font. Itâ€™s intended use is for body text and headers.

When paired with other bold text like the "JS" from Toronto JS' logo it should be used in medium weight to create visual contrast with the pairing text. The medium weight is also to be used for "main headings", where sufficient size difference exists between the body text and header.

For headers that are too close to text and the size difference is not enough to represent a "break" between text and header, the extra bold weight should be used.

On body text italic and bold text can be used to create emphasis when needed.

Source Sans Pro - Black

Used only for small details, like the "JS" in the Toronto JS logo or places where it will be the only piece of text present. This font intent is to draw attention to the text.

Source Code Pro - Regular
Used for code blocks and subtle emphasis on specific words and phrases. Like the "online" or "in person" mark on an event art.

Colour usage ðŸ”—

he main colour is "Toronto JS Red", it has two accent shades, one darker ("Breaking news") and one lighter ("CN Tower lights"). Those are the colours to be used in most contexts to highlight some details or make things "pop".

When using against a dark background the preferred pairing is the main colour follower by the lighter accent. Whereas the pairing for a light background should be the opposite, main colour followed by the darker accent.
